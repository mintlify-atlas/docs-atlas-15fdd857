The homepage displays two curated lists — **Projects** and **Works** — each rendered by a dedicated client component. Both components follow the same pattern: a hardcoded `data` array inside the component file, rendered as a list of clickable cards with hover highlights.

## Data structure

Each entry in both components uses the same shape:

```typescript
type Entry = {
  name: string;
  link?: string;        // optional — omit for entries with no public URL
  description: string;
};
```

When `link` is absent the card still renders but `href` falls back to `'#'`.

### Analytics event

Every project card fires a Rybbit analytics event on click:

```tsx
onClick={() => {
  if ('rybbit' in window) window.rybbit.event('project_link_click', {
    project_name: item.name,
    project_link: item.link
  });
}}
```

The event name is `project_link_click` and it carries `project_name` and `project_link` as properties.

## How to add or edit an entry

<Steps>
  <Step title="Open the component file">
    For a project, open `components/projects.tsx`. For a work entry, open `components/works.tsx`.
  </Step>
  <Step title="Edit the data array">
    Add a new object to the `data` array inside the component function, or modify an existing entry. Only `name` and `description` are required — `link` is optional.

    ```tsx
    {
      name: 'My New Project',
      link: 'https://example.com',
      description: 'A short description of what the project does.'
    }
    ```
  </Step>
  <Step title="Save and verify">
    The component re-renders automatically. Entries appear in the order they are defined in the array — no sorting is applied.
  </Step>
</Steps>

<Note>
  There is no external data source or CMS. All content lives directly in the component file. Changes require a code edit and a new deployment.
</Note>

## Render behaviour

Both components render each entry as a `<Link>` from `next-view-transitions`, giving the hover cards a view-transition animation when navigating. The highlight overlay is a pseudo-layer positioned behind each card that fades in on hover using Tailwind's `group-hover:opacity-100`.
